## 二、主控芯片 RV1106

### 2.1 芯片概述

RV1106 是瑞芯微（Rockchip）面向 AIoT 视觉场景推出的单核 SoC，集成了 CPU、NPU、ISP、编码器于一体，适合低功耗、小体积的 AI 摄像头应用。

### 2.2 处理器规格

| 模块 | 规格 |
|------|------|
| CPU | ARM Cortex-A7，主频 1.2GHz |
| NPU（G2） | 0.5 TOPS，支持 int4 / int8 / int16 量化 |
| NPU（G3） | 1.0 TOPS，支持 int4 / int8 / int16 量化 |
| ISP | 支持最大 5MP @ 30fps，具备 HDR、3DNR、WDR |
| 视频编码 | H.264 / H.265 硬件编码 |

### 2.3 通信接口规格

| 接口 | 数量 | 规格 |
|------|------|------|
| MIPI CSI | ≤2 路 | 每路 2-lane D-PHY v1.2，可合并为 4-lane |
| Display | 1 路 | RGB666（18-bit） |
| USB | 1 路 | USB 2.0 Host / Device |
| Ethernet | 1 路 | 10/100M + 内嵌 PHY |
| I2C | ≤5 路 | 100kbps 标准 / 400kbps 快速，7/10 位地址 |
| SPI | ≤2 路 | 主从模式，每路 2 片选 |
| UART | ≤6 路 | 5~8 位数据，最高 4Mbps |
| PWM | ≤12 路 | 32bit 定时器，支持捕获模式 |
| ADC | ≤2 路 | 10bit SAR-ADC，采样率 1MS/s |
| SDIO | ≤2 路 | SDIO 3.0，4-bit |
| I2S | ≤3 路 | 8TX/RX lanes，最高 192kHz，16~32bit |

## 三、当前摄像头：IMX415

### 3.1 传感器概述

Sony IMX415 是 STARVIS 系列背照式堆叠型 CMOS 传感器，专为 4K 安防监控、工业视觉等场景设计，具备优秀的低照度性能。

### 3.2 核心规格

| 参数 | 规格 |
|------|------|
| 制造商 | Sony |
| 型号 | IMX415-AAQR（彩色）/ IMX415-AAMR（黑白） |
| 传感器尺寸 | 1/2.8 英寸（对角线 6.43mm） |
| 有效像素 | 3864(H) × 2192(V)，约 8.46MP |
| 输出分辨率 | 3840 × 2160（4K UHD） |
| 像素尺寸 | 1.45µm × 1.45µm |
| 快门类型 | 卷帘快门（Rolling Shutter） |
| 最大帧率 | 90fps @ 10-bit，30fps @ 12-bit |
| 动态范围 | 支持 HDR（WDR） |
| 感光技术 | Sony STARVIS 背照式（BSI） |
| 输出接口 | MIPI CSI-2，4-lane |
| 供电电压 | 模拟 2.8V / 数字 1.1V / 接口 1.8V |
| 色彩深度 | 10-bit / 12-bit |

### 3.3 与 RV1106 ISP 的配合说明

- RV1106 ISP 支持最大 5MP @ 30fps，IMX415 以 4K 30fps 模式接入时接近 ISP 上限
- 建议在 NPU 推理场景下将输入流降为 1080P 或 720P，以降低处理延迟
- MIPI CSI 接入：使用 2-lane 模式时帧率受限，推荐使用 4-lane 以释放完整性能

## 四、后续升级摄像头：GC2093

### 4.1 传感器概述

GC2093 是格科微（Galaxycore）推出的 1/2.9 英寸 2MP CMOS 传感器，支持全高清输出，具有宽动态范围（WDR）和高帧率，广泛用于安防摄像头与嵌入式视觉系统。

### 4.2 核心规格

| 参数 | 规格 |
|------|------|
| 制造商 | Galaxycore（格科微） |
| 型号 | GC2093 |
| 传感器尺寸 | 1/2.9 英寸 |
| 有效像素 | 1920(H) × 1080(V)，约 2MP |
| 输出分辨率 | 1920 × 1080（Full HD） |
| 像素尺寸 | 2.8µm × 2.8µm |
| 快门类型 | 卷帘快门（Rolling Shutter） |
| 最大帧率 | 60fps @ 1080P |
| 动态范围 | 支持 WDR（宽动态范围） |
| 输出接口 | MIPI CSI-2 |
| 供电电压 | 2.8V / 1.8V / 1.2V |

### 4.3 与 RV1106 ISP 的配合说明

- GC2093 分辨率（2MP）完全在 RV1106 ISP 处理能力范围内，资源占用更低
- 1080P @ 60fps 可充分利用 RV1106 的视频编码能力
- 更低的数据量有利于提升 NPU 推理吞吐率
- 适合对实时性要求较高、分辨率需求不超过 1080P 的场景

## 五、摄像头对比

| 对比项 | IMX415（当前） | GC2093（后续升级） |
|--------|--------------|--------------|
| 分辨率 | 4K（3864×2192，8.46MP） | 1080P（1920×1080，2MP） |
| 传感器尺寸 | 1/2.8 英寸 | 1/2.9 英寸 |
| 像素尺寸 | 1.45µm | 2.8µm |
| 最大帧率 | 90fps（10-bit） | 60fps |
| 低照度性能 | 优秀（Sony STARVIS BSI） | 良好（WDR支持） |
| MIPI 接口 | CSI-2，4-lane | CSI-2 |
| ISP 负载 | 较高（接近 RV1106 上限） | 较低（余量充足） |
| NPU 推理适配 | 需降分辨率后推理 | 可直接 1080P 推理 |
| 适用场景 | 高分辨率采集、4K 存档 | 实时推理、低延迟应用 |
| 成本 | 较高 | 较低 |

**切换摄像头注意事项：**
- 切换至 GC2093 需重新适配 Linux 内核驱动（更新 device tree overlay）
- ISP 调校参数（IQ 文件）需针对 GC2093 重新标定
- RKNN 推理前处理管道中的分辨率、色彩格式参数需同步更新

## 六、接口与外设

### 6.1 登录与调试接口

| 接口 | 说明 |
|------|------|
| ADB | 通过 USB 连接，无需额外配置 |
| 串口（UART） | 波特率 115200，用于调试输出 |
| SSH | 需配置静态 IP 或通过 DHCP 获取地址 |

### 6.2 文件传输

支持通过 ADB / SCP（SSH）/ 串口 在开发板与 PC 之间传输文件。

### 6.3 开发环境

| 项目 | 说明 |
|------|------|
| 交叉编译 | Ubuntu 22.04，目标架构 ARMv7（arm-rockchip830-linux-uclibcgnueabihf） |
| SDK | Luckfox Pico RV1106 SDK |
| AI 框架 | RKNN Toolkit2 + RKNN Runtime |
| 多媒体 | RKMPI（Rockchip Media Process Interface） |
