# 烧录官方固件详细步骤

> 设备：Luckfox Pico Zero (RV1106)  
> 目标：烧录支持 GC2093 的官方完整固件  
> 日期：2026-07-07

---

## 一、准备工作

### 1.1 下载官方固件

**方式1：官方 Wiki 下载页**
- 访问：https://wiki.luckfox.com/zh/Luckfox-Pico-RV1106/Downloads
- 找到 "系统镜像" 部分
- 选择 **最新日期** 的固件（文件名末尾有日期，如 `Luckfox-xxx-240815.img`）
- 选择 Buildroot 版本（完整功能支持）

**方式2：百度网盘**
- 提取码：`2sf8`
- 选择适合 Pico Zero 的固件

**固件文件命名示例：**
```
Luckfox_Pico_Zero_Buildroot_xxx_240815.img
                                  ^^^^^^
                                  日期（始终选最新）
```

### 1.2 确认固件版本

选择固件时注意：
- ✅ **Buildroot** - 功能完整，带 RKIPC、多摄像头驱动支持（推荐）
- ⚠️ Ubuntu - 适合桌面使用，体积较大
- ⚠️ Minimal - 精简版，可能缺少部分驱动

**推荐下载：**
- `Luckfox_Pico_Zero_Buildroot_<最新日期>.img`

### 1.3 安装烧录工具（macOS）

**检查是否已安装：**
```bash
ls ~/Library/Application\ Support/upgrade_tool* 2>/dev/null
```

**下载并安装：**
```bash
# 下载 upgrade_tool（根据 macOS 版本）
# macOS 12.7: upgrade_tool_v2.25
# macOS 26.1: upgrade_tool_v2.44

# 解压并移动到 /usr/local/bin
cd ~/Downloads
unzip upgrade_tool_v2.44_mac.zip
cd upgrade_tool_v2.44_mac
sudo cp upgrade_tool /usr/local/bin/
sudo chmod +x /usr/local/bin/upgrade_tool

# 验证安装
upgrade_tool -v
# 输出：Program Data Ver:1.2.44
```

**或使用 SocToolKit（图形界面，需要 Windows）**

## 二、烧录步骤

### 2.1 进入烧录模式

**方法1：通过 ADB（当前方法）**
```bash
# 确认连接
adb devices

# 重启到烧录模式
adb -s cd48e940147caf5c reboot loader
```

**方法2：硬件按键（备用）**
1. 断开 USB 连接
2. 按住开发板上的 **BOOT 按键**
3. 保持按住，连接 USB 到电脑
4. 看到设备识别后松开 BOOT 键

### 2.2 确认进入 MaskRom 模式

```bash
# macOS 检查 USB 设备
system_profiler SPUSBDataType | grep -i rockchip

# 应显示类似：
# Product ID: 0x110b (MaskRom 模式)
# Vendor ID: 0x2207 (Rockchip)
```

### 2.3 执行烧录

```bash
# 进入固件所在目录
cd ~/Downloads

# 烧录固件（需要 sudo）
sudo upgrade_tool uf Luckfox_Pico_Zero_Buildroot_xxx_240815.img

# 烧录过程示例输出：
# Loading firmware...
# Support Type:RK110B  Loader ver:1.14  Loader Time:2024-01-15 09:00:00
# Upgrade firmware ok.
# Reset Device OK.
```

**烧录时间：** 约 2-5 分钟，取决于固件大小

### 2.4 等待完成

烧录完成后，开发板会自动重启。

**注意事项：**
- ⚠️ 烧录过程中**不要拔掉 USB**
- ⚠️ 不要中断电源
- ⚠️ 如果烧录失败，重新进入 MaskRom 模式再试

## 三、验证新固件

### 3.1 重新连接

```bash
# 等待开发板启动（约 20-30 秒）
adb devices

# 应显示设备已连接
```

### 3.2 检查系统版本

```bash
# 查看内核版本
adb shell uname -a

# 查看固件信息
adb shell cat /etc/issue
```

### 3.3 检查 GC2093 驱动

```bash
# 检查驱动模块目录
adb shell "find /lib/modules -name '*gc2093*' 2>/dev/null"

# 应输出类似：
# /lib/modules/5.10.160/kernel/drivers/media/i2c/gc2093.ko
```

### 3.4 连接摄像头并测试

**插入 GC2093 摄像头：**
1. 断开 USB 电源
2. 插入 GC2093 摄像头（金属面朝向 PCB 背面）
3. 重新连接电源

**检查识别状态：**
```bash
# 等待系统启动后
adb shell "i2cdetect -y 3 2>/dev/null || i2cdetect -y 4 2>/dev/null"
# 应在地址 0x37 看到设备

# 检查内核日志
adb shell "dmesg | grep gc2093"
# 应显示类似：
# gc2093 3-0037: driver version: 00.01.05
# gc2093 3-0037: gc2093 probe success

# 检查 RKIPC 配置
adb shell "cat /userdata/rkipc.ini | grep sensor_name"
# 应显示：sensor_name = gc2093
```

## 四、常见问题

### 4.1 无法进入 MaskRom 模式

**症状：**
- `upgrade_tool` 提示 "No devices found"
- 系统无法识别设备

**解决方法：**
1. **更换 USB 线缆** - 确保是数据线，非充电线
2. **更换 USB 端口** - 尝试不同的 USB 口
3. **检查驱动安装** - macOS 通常无需驱动，但确认系统设置 → 隐私与安全性中无阻止
4. **手动进入烧录模式**：
   - 完全断电
   - 按住 BOOT 键
   - 连接 USB（保持按住 BOOT）
   - 持续 3-5 秒后松开

### 4.2 烧录过程卡住或失败

**症状：**
- 进度条停止不动
- 报错 "Download Boot Fail"

**解决方法：**
1. **重新进入 MaskRom 模式**
2. **检查固件文件完整性**：
   ```bash
   # 检查文件大小（应为几百 MB）
   ls -lh Luckfox_Pico_Zero_Buildroot_*.img
   ```
3. **重新下载固件** - 可能下载损坏
4. **尝试使用 Windows + SocToolKit** - 图形界面更稳定

### 4.3 烧录后无法启动

**症状：**
- ADB 无法连接
- 串口无输出
- 指示灯不亮

**解决方法：**
1. **重新烧录** - 可能烧录未完成
2. **检查启动介质配置** - 确认固件是 eMMC 版本
3. **尝试不同版本固件** - 选择更早或更新的版本

### 4.4 烧录后找不到 GC2093 驱动

**症状：**
- 固件正常启动
- 但 `/lib/modules` 中没有 gc2093.ko

**可能原因：**
- 下载了精简版固件
- 选择了不包含摄像头驱动的版本

**解决方法：**
1. 重新下载 **Buildroot 完整版** 固件
2. 确认固件名称包含 "RKIPC" 或 "IPC"
3. 或尝试自行编译 SDK

## 七、备份当前固件（可选）

如果需要保留当前固件：

```bash
# 读取当前固件（需在 MaskRom 模式下）
sudo upgrade_tool rd 0 0x10000 backup.img

# 注意：此操作需要较长时间
```

**建议：** 如果当前固件有重要配置，先备份 `/userdata` 目录：
```bash
adb pull /userdata ./userdata_backup
```
