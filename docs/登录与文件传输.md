## 二、ADB 登录

### Windows

1. 下载 ADB 工具包，将路径加入系统环境变量
2. 打开 CMD，执行：
   ```bash
   adb shell
   ```
3. 多设备时先查看序列号再指定：
   ```bash
   adb devices
   adb -s <序列号> shell
   ```

### Ubuntu

```bash
sudo apt-get install android-tools-adb -y
# 查找设备 ID（通常为 2207:0019）
lsusb
# 创建 udev 规则
sudo vim /etc/udev/rules.d/51-android.rules
# 写入：SUBSYSTEM=="usb", ATTR{idVendor}=="2207", ATTR{idProduct}=="0019", MODE="0666", GROUP="plugdev"
sudo chmod 644 /etc/udev/rules.d/51-android.rules
sudo udevadm control --reload-rules
# 重新插拔设备后连接
adb shell
```

## 三、SSH 登录

### Windows — USB 静态 IP

1. 关闭 Windows 防火墙
2. 找到 RNDIS 网卡（名称含 "Remote NDIS"），手动设置 IPv4 为 `172.32.0.100`
3. 用 MobaXterm → Session → SSH，输入静态 IP `172.32.0.93`（Buildroot）或 `172.32.0.70`（Ubuntu）

### Windows — 路由器动态 IP

1. 网线连接开发板与路由器
2. 用 Zenmap 执行 Ping Scan，对比上电前后结果找到板子 IP
3. MobaXterm SSH 连接该 IP

### Ubuntu — USB 静态 IP

```bash
# 确认 RNDIS 注册成功
sudo dmesg | tail -n 50
# 查看新出现的网卡名（如 enxdee6de754a7f）
ip link
# 配置接口
sudo ip addr add 172.32.0.98/24 dev <网卡名>
sudo ip link set <网卡名> up
# 然后 SSH 连接
ssh root@172.32.0.93
```

## 四、串口登录（推荐用于调试）

接线：USB 转 TTL 模块 → 开发板 TX、RX、GND

1. 设备管理器查看 COM 口编号
2. MobaXterm → Session → Serial，选择 COM 口，波特率 `115200`
3. 按回车后输入用户名/密码

> 串口可获取完整启动日志，开发阶段强烈推荐。

## 五、文件传输

### ADB 传输

```bash
# 本地 → 开发板
adb push test/ /root

# 开发板 → 本地
adb pull /home/luckfox/1.txt .
```

### SCP 传输

```bash
# 单文件上传
scp luckfox.txt root@<板子IP>:/root

# 目录上传
scp -r luckfox/ root@<板子IP>:/root

# 从板子下载文件
scp root@<板子IP>:/root/luckfox.txt .

# 从板子下载目录
scp -r root@<板子IP>:/root/luckfox .
```

### SFTP 图形化传输

使用 electerm（免费开源，支持 Linux/Mac/Windows），通过 SSH 连接后可图形化拖拽传输文件。

### 大文件/频繁更新

```bash
# rsync 只同步差异部分，节省带宽
rsync -avz local_dir/ root@<板子IP>:/root/remote_dir/
```

## 六、Samba 共享（Ubuntu ↔ Windows）

**Ubuntu 端：**
```bash
sudo apt-get install samba -y
sudo vim /etc/samba/smb.conf   # 末尾追加共享配置
sudo smbpasswd -a luckfox      # 设置共享密码
ifconfig                       # 查看 IP
```

**Windows 端：** 地址栏输入 `\\<Ubuntu IP>\share`，用设置的账户密码登录。
