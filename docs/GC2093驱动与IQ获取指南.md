## 二、驱动编译（Docker 方案，已验证）

官方固件 250802 不包含 gc2093.ko，需要手动编译。

### 2.1 为什么用 Docker

- **环境隔离**：避免污染主机环境
- **可复现**：所有依赖在容器内固定
- **跨平台**：在 macOS/Windows 上也能编译 ARM 内核模块

### 2.2 关键约束

| 约束项 | 值 | 说明 |
|--------|-----|------|
| kernel vermagic | `5.10.160 mod_unload ARMv7 thumb2 p2v8` | 必须完全匹配板上内核 |
| 交叉编译器 | `arm-linux-gnueabihf-gcc` (apt) | **不需要** Rockchip 私有 toolchain |
| CONFIG_MODVERSIONS | 未启用 | 无需 CRC 符号校验 |
| Python 环境 | `python` 命令必须存在 | Ubuntu 22.04 需要 `python-is-python3` |

### 2.3 构建 Docker 镜像

**Dockerfile**（保存为 `/tmp/gc2093-build/Dockerfile`）：

```dockerfile
FROM ubuntu:22.04
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y \
    gcc-arm-linux-gnueabihf \
    bc flex bison libssl-dev libelf-dev \
    make git wget python3 python-is-python3 cpio rsync \
    && rm -rf /var/lib/apt/lists/*
WORKDIR /build
```

**构建镜像**：
```bash
docker build -t rv1106-builder:latest /tmp/gc2093-build/
```

> ⚠️ **关键**：必须安装 `python-is-python3`，否则内核构建会失败（内核脚本调用 `python`，Ubuntu 22.04 默认只有 `python3`）。

### 2.4 编译脚本

**保存为 `/tmp/gc2093-build/compile.sh`**：

```bash
#!/bin/bash
set -e

echo "=== [1/5] 初始化 git sparse-checkout ==="
mkdir -p /build/kernel-src && cd /build/kernel-src
git init
git remote add origin https://github.com/LuckfoxTECH/luckfox-pico.git
git config core.sparseCheckout true

echo "=== [2/5] Fetch (depth=1, blob:none) ==="
echo "sysdrv/source/kernel" > .git/info/sparse-checkout
git fetch --depth=1 --filter=blob:none origin main

echo "=== [3/5] Sparse-checkout 内核源码 ==="
git checkout main

KDIR=/build/kernel-src/sysdrv/source/kernel
echo "内核版本: $(head -5 $KDIR/Makefile | grep -E 'VERSION|PATCHLEVEL|SUBLEVEL')"

echo "=== [4/5] 配置内核 + modules_prepare ==="
cd $KDIR
make ARCH=arm luckfox_rv1106_linux_defconfig
./scripts/config --set-val CONFIG_VIDEO_GC2093 m
make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- modules_prepare -j$(nproc)

echo "=== [5/5] 编译 gc2093.ko ==="
make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- \
    M=drivers/media/i2c \
    CONFIG_VIDEO_GC2093=m \
    modules -j$(nproc) 2>&1

# 验证结果
KO=$KDIR/drivers/media/i2c/gc2093.ko
if [ -f "$KO" ]; then
    cp "$KO" /output/gc2093.ko
    echo "=== 编译成功！==="
    arm-linux-gnueabihf-objdump -p /output/gc2093.ko | grep "VERNEEDNUM\|vermagic" || true
    strings /output/gc2093.ko | grep vermagic
else
    echo "ERROR: gc2093.ko 未找到"
    exit 1
fi
```

### 2.5 执行编译

```bash
mkdir -p /tmp/gc2093-output

docker run --rm \
  -v /tmp/gc2093-build/compile.sh:/build/compile.sh:ro \
  -v /tmp/gc2093-output:/output \
  rv1106-builder:latest \
  bash /build/compile.sh
```

### 2.6 验证编译结果

```bash
# 检查 vermagic
strings /tmp/gc2093-output/gc2093.ko | grep vermagic
# 期望输出：vermagic=5.10.160 mod_unload ARMv7 thumb2 p2v8

# 检查文件大小
ls -lh /tmp/gc2093-output/gc2093.ko
# 约 254KB
```

## 三、IQ 文件获取

### 3.1 从 SDK 提取 IQ 文件

**位置**：`media/isp/release_camera_engine_rkaiq_rv1106_arm-rockchip830-linux-uclibcgnueabihf/isp_iqfiles/`

**方式一：克隆 SDK（推荐）**

```bash
# 浅克隆（节省空间）
git clone --depth=1 https://gitee.com/LuckfoxTECH/luckfox-pico.git /tmp/luckfox-sdk

# 提取 IQ 文件
cp /tmp/luckfox-sdk/media/isp/release_camera_engine_rkaiq_rv1106_arm-rockchip830-linux-uclibcgnueabihf/isp_iqfiles/gc2093*.json \
   /tmp/gc2093-output/
```

**方式二：直接下载单个文件**

```bash
wget -O /tmp/gc2093-output/gc2093_SIDA209300461_60IRC_F20.json \
  https://gitee.com/LuckfoxTECH/luckfox-pico/raw/main/media/isp/release_camera_engine_rkaiq_rv1106_arm-rockchip830-linux-uclibcgnueabihf/isp_iqfiles/gc2093_SIDA209300461_60IRC_F20.json
```

### 3.2 IQ 文件命名规则（关键！）

rkipc 根据 DTS 中的摄像头模组信息**动态拼接** IQ 文件名：

```
{sensor}_{rockchip,camera-module-name}_{rockchip,camera-module-lens-name}.json
```

**示例**：如果 DTS 中配置为：
```dts
rockchip,camera-module-name = "LUCKFOX-GC2093";
rockchip,camera-module-lens-name = "30IRC-F20";
```

则 rkipc 期望的文件名为：
```
gc2093_LUCKFOX-GC2093_30IRC-F20.json
```

SDK 原始文件名通常为 `gc2093_SIDA209300461_60IRC_F20.json`，需要**复制并改名**或创建符号链接。

### 3.3 IQ 文件内容

| 参数类别 | 包含内容 |
|---------|---------|
| AWB（白平衡） | 色温增益、色彩校正矩阵 |
| AEC（自动曝光） | 曝光时间、增益范围 |
| Demosaic | 去马赛克插值算法 |
| Denoise | 2DNR/3DNR 降噪参数 |
| Gamma | 伽马曲线 |
| Sharpness | 锐化边缘增强 |
| LSC | 镜头阴影校正（渐晕） |

## 四、部署到开发板

### 4.1 部署 gc2093.ko

```bash
# 传输到板子
adb push /tmp/gc2093-output/gc2093.ko /oem/usr/ko/gc2093.ko

# 修改 insmod_ko.sh，添加 gc2093 到加载列表
adb shell "vi /oem/usr/ko/insmod_ko.sh"
# 在 mia1321.ko 前添加：__insmod gc2093.ko
# rmmod 列表添加 gc2093
```

### 4.2 部署 IQ 文件

```bash
# 部署原始文件
adb push /tmp/gc2093-output/gc2093_SIDA209300461_60IRC_F20.json \
         /oem/usr/share/iqfiles/

# 创建 rkipc 期望的命名（与 DTS 模组名对应）
adb shell "cp /oem/usr/share/iqfiles/gc2093_SIDA209300461_60IRC_F20.json \
              /oem/usr/share/iqfiles/gc2093_LUCKFOX-GC2093_30IRC-F20.json"

# 验证
adb shell "ls -lh /oem/usr/share/iqfiles/gc2093*.json"
```

## 五、验证驱动加载

```bash
# 1. 检查模块加载
adb shell "lsmod | grep gc2093"
```

## 六、常见问题

### ❌ 问题：编译时报错 `/usr/bin/env: 'python': No such file or directory`

**原因**：Ubuntu 22.04 默认只有 `python3`，内核构建脚本调用 `python` 命令。

**解决**：在 Dockerfile 中添加 `python-is-python3` 包。

---

### ❌ 问题：insmod 时报错 `Invalid module format`

**现象**：
```bash
insmod: can't insert 'gc2093.ko': invalid module format
```

**原因**：vermagic 不匹配。

**排查**：
```bash
# 板上内核 vermagic
cat /proc/version
modinfo gc2093.ko | grep vermagic

# 对比是否完全一致
```

**解决**：确保编译时使用 `luckfox_rv1106_linux_defconfig`，且板上固件与编译源 kernel 版本一致。

---

### ❌ 问题：rkipc 启动时报错 `access /oem/usr/share/iqfiles/gc2093_XXX.json failed`

**原因**：IQ 文件名与 DTS 中 `camera-module-name` + `camera-module-lens-name` 拼接不一致。

**解决**：
1. 查看 DTS 中 gc2093 节点的 `rockchip,camera-module-name` 和 `rockchip,camera-module-lens-name`
2. 按格式 `gc2093_{module-name}_{lens-name}.json` 重命名或复制 IQ 文件

---

### ❌ 问题：gc2093.ko 加载成功但不 probe

**现象**：`lsmod` 显示 gc2093 已加载，但 `dmesg | grep gc2093` 无 probe 成功信息。

**原因**：DTB 中没有 gc2093 节点，或 `compatible` 字段不匹配 `"galaxycore,gc2093"`。

**解决**：检查 DTB 是否正确修改并生效（参考《GC2093完整配置指南.md》第六章）。

## 七、技术细节

### 7.1 驱动关键参数

| 参数 | 值 |
|------|-----|
| I2C 地址 | 0x37（7位）/ 0x6e（8位） |
| MCLK | 24MHz |
| 分辨率 | 1920×1080 |
| 输出格式 | RAW Bayer RGGB 10bit |
| MIPI 接口 | CSI-2，2-lane（推荐）或 4-lane |

### 7.2 git sparse-checkout 优势

完整 SDK 约 3GB，sparse-checkout 只拉取内核部分（约 600MB），节省：
- 下载时间：~75%
- 磁盘占用：~80%

命令详解：
```bash
git config core.sparseCheckout true
echo "sysdrv/source/kernel" > .git/info/sparse-checkout
git fetch --depth=1 --filter=blob:none origin main
```

`--filter=blob:none` 进一步延迟加载大文件，仅在 checkout 时按需下载。
