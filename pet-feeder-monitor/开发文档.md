# pet-feeder-monitor 开发文档

> 项目用途：宠物智能喂食器测试监控工具  
> 功能：RTSP 推流画面预览 + 推理结果实时显示  
> 更新日期：2026-07-04

---

## 一、项目概述

本项目是宠物智能喂食器的 PC 端测试工具，用于开发调试阶段：
- 实时预览设备 RTSP 推流画面
- 显示宠物检测框和个体 ID
- 显示食物余量状态
- 手动触发喂食指令（测试 UART → 电机链路）
- 本地记录识别事件日志

**不是**生产环境 APP，仅用于开发测试。

## 五、MQTT 消息格式

### 5.1 订阅：推理结果（设备 → 工具）

Topic：`feeder/results`

```json
{
    "ts": 1720000000,
    "pets": [
        {
            "id": "cat_01",
            "confidence": 0.92,
            "box": [320, 400, 180, 160],
            "eating": true
        }
    ],
    "food_level": "half"
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| ts | int | Unix 时间戳 |
| pets | array | 检测到的宠物列表（可为空） |
| pets[].id | string | 宠物 ID（注册时分配） |
| pets[].confidence | float | 置信度 0~1 |
| pets[].box | array | [x, y, w, h] 像素坐标 |
| pets[].eating | bool | 是否判定为进食中 |
| food_level | string | full / half / low / empty |

### 5.2 发布：控制指令（工具 → 设备）

Topic：`feeder/cmd`

手动触发喂食：
```json
{"act": "feed", "source": "manual_test"}
```

## 六、核心模块说明

### 6.1 mqtt_client.py

```python
import paho.mqtt.client as mqtt
import json, threading
from config import MQTT_HOST, MQTT_PORT, MQTT_TOPICS

class MQTTClient:
    def __init__(self):
        self.results = {}
        self.lock = threading.Lock()
        self.client = mqtt.Client()
        self.client.on_message = self._on_message
        self.client.connect(MQTT_HOST, MQTT_PORT)
        self.client.subscribe(MQTT_TOPICS["results"])
        self.client.loop_start()

    def _on_message(self, client, userdata, msg):
        with self.lock:
            self.results = json.loads(msg.payload)

    def get_results(self):
        with self.lock:
            return self.results.copy()

    def trigger_feed(self):
        payload = json.dumps({"act": "feed", "source": "manual_test"})
        self.client.publish(MQTT_TOPICS["cmd"], payload)
```

### 6.2 overlay.py

```python
import cv2

COLORS = {
    "eating":     (0, 255, 0),    # 绿色：进食中
    "detected":   (0, 165, 255),  # 橙色：检测到但未进食
    "food_low":   (0, 0, 255),    # 红色：食物不足
    "food_ok":    (255, 255, 255) # 白色：食物充足
}

def draw(frame, results):
    # 宠物检测框
    for pet in results.get("pets", []):
        x, y, w, h = pet["box"]
        color = COLORS["eating"] if pet.get("eating") else COLORS["detected"]
        cv2.rectangle(frame, (x, y), (x+w, y+h), color, 2)
        label = f"{pet['id']} {pet['confidence']:.0%}"
        cv2.putText(frame, label, (x, y-8),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.6, color, 2)

    # 食物余量
    level = results.get("food_level", "--")
    color = COLORS["food_low"] if level in ("low", "empty") else COLORS["food_ok"]
    cv2.putText(frame, f"Food: {level}", (10, 35),
                cv2.FONT_HERSHEY_SIMPLEX, 1.0, color, 2)

    # 时间戳
    import time
    cv2.putText(frame, time.strftime("%H:%M:%S"), (10, frame.shape[0]-10),
                cv2.FONT_HERSHEY_SIMPLEX, 0.6, (200, 200, 200), 1)
    return frame
```

### 6.3 main.py

```python
import cv2, time, os
from config import RTSP_URL, WINDOW_TITLE, LOG_ENABLED, LOG_DIR
from mqtt_client import MQTTClient
from overlay import draw

def main():
    mqtt = MQTTClient()
    cap  = cv2.VideoCapture(RTSP_URL)

    if LOG_ENABLED:
        os.makedirs(LOG_DIR, exist_ok=True)
        log_file = open(f"{LOG_DIR}/{time.strftime('%Y%m%d_%H%M%S')}.log", "a")

    print("启动成功。按 [空格] 手动触发喂食，按 [Q] 退出。")

    while True:
        ret, frame = cap.read()
        if not ret:
            print("RTSP连接断开，重连中...")
            cap = cv2.VideoCapture(RTSP_URL)
            continue

        results = mqtt.get_results()
        frame   = draw(frame, results)
        cv2.imshow(WINDOW_TITLE, frame)

        key = cv2.waitKey(1) & 0xFF
        if key == ord('q'):
            break
        elif key == ord(' '):
            mqtt.trigger_feed()
            print(f"[{time.strftime('%H:%M:%S')}] 手动触发喂食")
            if LOG_ENABLED:
                log_file.write(f"{time.strftime('%H:%M:%S')} MANUAL_FEED\n")

        # 记录识别事件
        if LOG_ENABLED and results:
            log_file.write(f"{time.strftime('%H:%M:%S')} {results}\n")
            log_file.flush()

    cap.release()
    cv2.destroyAllWindows()

if __name__ == "__main__":
    main()
```

## 八、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 画面打不开 | RTSP地址错误或设备未推流 | 确认设备IP，先用VLC验证RTSP地址 |
| 无识别结果显示 | MQTT未连接或Topic不一致 | 检查config.py中MQTT_HOST和Topic |
| 画面延迟高 | 网络问题或VLC缓冲 | 确认与设备在同一局域网 |
| MQTT连接失败 | 服务器未启动 | 先启动MQTT Broker（如Mosquitto） |

from flask import Flask, Response
app = Flask(__name__)

@app.route('/video')
def video():
    def generate():
        while True:
            ret, frame = cap.read()
            frame = draw(frame, mqtt.get_results())
            _, buf = cv2.imencode('.jpg', frame)
            yield (b'--frame\r\nContent-Type: image/jpeg\r\n\r\n'
                   + buf.tobytes() + b'\r\n')
    return Response(generate(),
                    mimetype='multipart/x-mixed-replace; boundary=frame')
